Pass by Reference vs. Pass by Value

28 July 2015

JavaScript is both a pass by reference and pass by value language. When dealing with primitive types of data, JavaScript will pass that data by value but will pass JavaScript Objects by reference. This inconsistency can lead to confusion for new developers who may not be getting the results they expect when using JavaScript objects or arrays (which are also JS objects). Let's explore this aspect of the language and the implications it has on your code.

Passing

Passing typically refers to invoking a function with arguments. You "pass" arguments into a function at call-time. 

var sum = function(x, y) {  
  return x + y;
}

var a = 2;  
var b = 3;

sum(a, b); // Passing arguments into the sum function

For this lesson, passing can also refer to assigning a value to a variable. 

var a = 5;  
var b = a;

console.log(b); // 5
Passing by Value

First we'll cover passing by value. Passing by value essentially means creating a new copy of a piece of data in memory so any changes made to the new variable (or parameter in a function) does not affect the original piece of data. Let's explore what that means. 

var a = 3;  
var b = a;  
a = 5;

console.log(b); // 3

As you can see, we passed the value of a to b when we declared the new variable b. We then went on to change the value of a after passing its value to b and the value of b was unchanged.

This is because when we assign the value of a variable to a new variable, if the value is a primitive type in JavaScript, a copy of that data is made. Primitive types in JavaScript are: undefined null boolean number string.

Here's an example of passing by value into a function. 

var sumSquares = function(x, y) {  
  x = x * x;
  y = y * y;
  return x + y;
}

var a = 2;  
var b = 3;

sumSquares(a, b);  // 13  
console.log(a);  // 2  
console.log(b);  // 3

Here we're initializing variables a and b to the values 2 and 3 respectively. We then pass those variables into our function sumSquares which goes on to manipulate those values by squaring them and then returning the sum of the squared values. After the function runs, the original values of a and b are unchanged. This is because 2 and 3 are numbers which are primitive types in JavaScript. Now let's take a look at passing by reference.

Passing by Reference

The alternative to passing by value is passing data by reference. In JavaScript, objects are always passed by reference. That means that instead of making a copy of the data when it is passed, JavaScript will instead store a reference to that data. In other words, when you pass an object, a pointer to that object's place in memory is created and assigned to the new variable or parameter on the other end of the pass. Let's look at the implications of passing by reference. 

var passByRef = function(obj) {  
    // inside this function, obj is just a pointer to myObject
    // any changes made to obj inside this function will affect myObject
    obj.name: “Susan”;
    return obj;
}

var myObject = {  
    name: “Travis”,
    age: 27
}

passByRef(myObject);  
console.log(myObject); // { name: “Susan”, age: 27 };

What happened here? As we were discussing, myObject was passed by reference to the function passByRef. What that means is that the parameter obj inside the function is just a pointer to where myObject lives in memory, therefore any changes made to the parameter obj will be directed back to the original object. When a function performs actions that affect the variables outside of the function, that function is said to have side-effects. A function that has side-effects does not follow code design best practices and should be avoided whenever possible. Here's another example that may leave you scratching your head. 

var myObj = {  
  name: 'Travis'
};

var newObj = myObj;

newObj.name = 'Sarah';

console.log(newObj); // { name: 'Sarah' };  
console.log(myObj);  // { name: 'Sarah' };

newObj is also just a pointer to the original object myObj so any changes we make to either variable will be reflected in both variables. The object only exists in one place in memory and every time we pass that object, we simply create a pointer to the original object.

How can we avoid side effects then? An easy and generalized answer that does not apply to every situation is to create a new object/array within the function that is being called and place the mutated values into that new object. 

var firstSquareRoot = function(arr) {  
    var newArray = [];  // Create a new array so we don't modify myArray
    for (var i = 0; i < arr.length; i++) {
        newArray.push(arr[i] * arr[i]);  // Push mutated values into newArray
    }

    return newArray;  // Return the newArray
                      // We never changed the values in arr
}

var myArray = [1, 2, 3];  
var squaredArray = firstSquareRoot(myArray);

console.log(squaredArray);    // [1, 4, 9]  
console.log(myArray);         // [1, 2, 3] // Original array is unmodified
Conclusion

Knowing when data is passed by value and when it's passed by reference is simple knowledge that will save you many headaches when debugging code that would otherwise look like it had a mind of its own. If you simply remember that passing an object to a function or assigning it to a new variable passes it by reference, you'll be a better developer due to writing functions without side-effects and avoiding bugs.
