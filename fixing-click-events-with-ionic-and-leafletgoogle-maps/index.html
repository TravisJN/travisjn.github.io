Fixing click events with Ionic and Leaflet/Google Maps

19 May 2015

This short post is meant to solve a scarcely documented problem that occurs when using Ionic with a mapping library such as Leaflet or Google Maps. The bug is such that click or touch events on the map that perform actions such as opening marker popups or registering the position of the mouse stop firing when the user clicks an element that isn't the map. It appears then as if all of the map's event listeners are removed whenever a touch or click event occurs off the map.

Fortunately it's an easy fix and has to do with a built-in feature of Ionic that changes the default properties of touch events so they feel consistent across different devices. Some mobile browsers implement a 300ms delay from the time a touch event occurs to when the touch event executes in order to listen for double-taps. Ionic removes this delay. This is fine for most apps but causes an issue when a part of your app, such as a map library, also decides to start messing with the default behavior of click events.

Due to the nature of mapping clients, the default touch behavior has to be changed on mobile devices. Thus the conflicting touch settings interfere with one another and your click events stop working properly. So how do we fix it?

Enter Ionic's simple to implement data-tap-disabled tag. All you have to do is wrap the div you attached your map to in another div with the attribute data-tap-disabled="true". That should do it, happy coding!

Example:
<div data-tap-disabled="true"> <div id="google-map"></div> </div>

Source:
http://ionicframework.com/docs/api/page/tap/
